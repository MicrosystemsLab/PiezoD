piezoD - download

piezoD is (and will always be) free and open source software. If you download and use it, we only ask that you cite the first paper that we wrote on the subject, entitled “Design optimization of piezoresistive cantilevers for force sensing in air and water”.

We have written several followup papers on piezoresistor design and fabrication (e.g. 1, 2 and 3). You can find additional papers that we've written here. For more details please also see Advances in high bandwidth nanomechanical force sensors with integrated actuation.

piezoD is composed of several Matlab classes. You can download official releases from here.

  • v0.7.1 - 01/09/13 - 1) Added the helper scripts for ion implantation table generation, which were accidentally not included in v0.6.0 and 2) Fixed a minor units bug in cantileverImplantation.m that affected plotting the resulting dopant concentration profile.

  • v0.7.0 - 07/15/12 - Several minor changes: 1) Switched from using the interior point method to SQP for optimization. 2) Fixed ion implantation sheet resistance calculations. 3) Several small bug fixes and speed improvements throughout the code.

  • v0.6.0 - 06/12/12 - Added ion implantation modeling and optimization via a lookup table tabulated from TSuprem simulations. Also reintegrated the poly and piezoelectric classes to the release and cleaned up the code.

  • v0.5.0 - 01/16/11 - Several major changes: 1) temperature dependent modeling of piezoresistor self-heating, 2) better mechanics modeling for segmented cantilevers, particularly resonant frequency via the Rayleigh-Ritz method, 3) a Simulink model to investigate time-domain response, 4) cleaned up the code.

  • v0.4.2 - 07/06/10 - More big changes. Added optional code to simulate thermal or piezoelectric actuation at the base of the cantilever. Optional features now include actuation, tip mass, and the number of piezoresistors in the Wheatstone bridge. Also fixed some bugs from the last release.

  • v0.4.0 - 06/17/10 - Big changes. Wrote a piezoelectric sensor class. Added thermal modeling to PR cantilevers and temperature constrained design. Lots of performance improvements.

  • v0.3.1 - 03/30/10 - Added amplifier current noise, changed some default optimization constraints and the default gamma value. Added a parameter for the number of silicon resistors in the bridge.

  • v0.3.0 - 09/13/09 - Rewrote the method for passing in nonlinear constraints to the optimizer. It is much easier to optimize for a spring constant value or range now. Also changing the optimization goal from force to displacement can be specified more easily.

  • v0.2.1 - 07/24/09 - Fixed several bugs in the diffusion piezoresistor code and typos in sample_code.m

  • v0.2.0 - 04/28/09 - Rewrote constraint handling, other important things

  • v0.1.0 - 03/01/09 - Initial release

The versions posted here will lag behind the actual code development. The most recent code and detailed changes can be found at the github page.

piezoD requires Matlab R2008a or later and the optimization toolbox. Object oriented features were introduced in R2008a which are used. Installation is just a matter of placing the following files in your Matlab path.

  • cantilever.m

  • cantileverDiffusion.m

  • cantileverEpitaxy.m

  • cantileverImplantation.m

  • cantileverPiezoelectric.m

  • cantileverPoly.m

  • lookupTable.mat

You can check that it works by running sampleCode.m. Please request any new features and report bugs here.
